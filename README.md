# テレアポリスト生成ツール

広告代理店の営業チーム向けリード生成ツール。Google検索のリスティング広告（スポンサー枠）を自動検索し、広告主のURL・電話番号・代表者名を抽出してGoogle Spreadsheetに出力する。

## 機能

- **業種 + エリア/駅名のローラー検索**: 「美容院」+「五反田駅」「恵比寿駅」...のように業種キーワードと複数の場所を指定して一括検索
- **リスティング広告の自動検出**: Google検索結果からスポンサー広告のURLを抽出（広告クリックは行わず、DOM読み取りのみ）
- **位置エミュレーション検索**: 47都道府県 + 20政令指定都市の位置情報を指定して、その地域の広告を取得（X-Geoヘッダーによるジオターゲティング）
- **複数ページ広告取得**: 検索結果の1〜5ページ目まで広告を取得可能（ドメイン重複排除付き）
- **AI連絡先抽出**: 広告主サイトをスクレイピングし、Gemini 2.0 Flash で電話番号・代表者名・会社名を構造化抽出
- **3段階抽出**: LP → 同ドメイン内会社概要ページ → Google検索補完の順に電話番号・会社名を探索
- **地域優先電話番号選択**: 検索地域の市外局番に一致する固定電話を優先。フリーダイヤル（0120/0800）より地域の固定電話を優先して抽出
- **リアルタイム進捗表示**: SSE (Server-Sent Events) でジョブの進行状況をリアルタイムにUI表示
- **Google Spreadsheet出力**: ブラウザでGoogle OAuth認証 → ワンクリックでスプレッドシートにエクスポート。エクスポート済みリンクはアプリ内に保存
- **Spreadsheet追記モード**: 過去にエクスポートしたスプレッドシートを記憶。2回目以降は既存データの下に追記。A列に日付、B列に時刻を記録

## 技術スタック

| カテゴリ | 技術 |
|---------|------|
| フレームワーク | Next.js 16 (App Router) / React 19 / TypeScript |
| スタイリング | Tailwind CSS v4 (PostCSS) |
| DB | SQLite (better-sqlite3) + Drizzle ORM |
| スクレイピング | Playwright (CDP接続でリアルChrome使用) |

## セットアップ

### 1. 依存パッケージのインストール

```bash
npm install
```

### 2. 環境変数の設定

`.env.example` を `.env.local` にコピーして値を設定:

```bash
cp .env.example .env.local
```

| 変数名 | 用途 | 必須 |
|--------|------|------|
| `GEMINI_API_KEY` | Gemini API (連絡先抽出) | 必須 |
| `GOOGLE_CLIENT_ID` | Google OAuth (Sheets出力) | Sheets出力時のみ |
| `GOOGLE_CLIENT_SECRET` | Google OAuth (Sheets出力) | Sheets出力時のみ |
| `NEXT_PUBLIC_BASE_URL` | OAuthコールバックURL | デフォルト: `http://localhost:3000` |

### 3. Google OAuth の設定

Spreadsheetエクスポート機能を使う場合:

1. [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
2. **APIとサービス → ライブラリ** で「Google Sheets API」を有効化
3. **APIとサービス → 認証情報** → **OAuth クライアント ID** を作成（ウェブ アプリケーション）
4. **承認済みのリダイレクト URI** に `http://localhost:3000/api/auth/google/callback` を追加
5. **OAuth 同意画面** でテストユーザーにアカウントを追加
6. クライアントIDとシークレットを `.env.local` に設定

### 4. Chromeをデバッグモードで起動

Playwrightはリアルなブラウザに接続するため、Chromeをリモートデバッグ付きで起動する必要がある:

```bash
open -na "Google Chrome" --args --remote-debugging-port=9222 --user-data-dir=/tmp/chrome-debug
```

### 5. DBセットアップ

```bash
npx drizzle-kit push
```

### 6. 開発サーバー起動

```bash
npm run dev
```

http://localhost:3000 でアクセス。

## 使い方

1. ダッシュボードで業種キーワード（例: 「ジム」「美容院」）を入力
2. 検索モードを選択:
   - **キーワードに地名を含めて検索**: 「路線から選ぶ」または「エリアから選ぶ」でロケーションを選択。検索クエリに地名が追加される（例: 「ジム 渋谷駅」）
   - **位置情報を指定して検索**: 都道府県・政令指定都市を選択。X-Geoヘッダーで位置をエミュレートし、その地域の広告を取得
3. 検索ページ数を設定（1〜5ページ、デフォルト1）
4. 「検索開始」でジョブを実行
5. リアルタイムで進捗を確認。完了後、結果テーブルにURL・電話番号・代表者名が表示される
6. 「Googleアカウントで認証」→ Spreadsheet URLを入力 →「Spreadsheetに出力」

## アーキテクチャ

```
ユーザー入力 (キーワード + 場所 + ページ数)
    ↓
POST /api/jobs → DBにジョブ作成
    ↓
search-orchestrator (バックグラウンド実行)
    ├── Google検索 (Playwright CDP)
    │   ├── キーワード検索: 「ジム 渋谷駅」のようにクエリに地名追加
    │   └── 位置エミュレーション: X-Geoヘッダーでジオターゲティング
    ├── 1〜5ページの広告URLをDOM読み取りで抽出（ドメイン重複排除）
    ├── 広告主サイトに直接アクセス → テキスト抽出
    ├── Gemini AI → 電話番号・代表者名を構造化抽出
    ├── DB保存 + SSEでリアルタイム通知
    └── 次の検索へ (3-8秒のランダム遅延)
    ↓
GET /api/jobs/[id]/stream (SSE) → フロントエンドに進捗配信
    ↓
POST /api/jobs/[id]/export → Google OAuth認証 → Spreadsheetに出力/追記
```

## 電話番号の選択ロジック

ページ内に複数の電話番号がある場合、以下の優先順位で選択する:

1. **検索地域の市外局番に一致する固定電話**（例: 福岡県検索 → 092-xxx を最優先）
2. **任意の固定電話**（地域は異なるがフリーダイヤルではない番号）
3. **フリーダイヤル**（0120/0800。他に番号がない場合のみ）

市外局番と都道府県のマッピングにより、47都道府県 + 政令指定都市レベルで地域判定を行う。

## 検索モードの違い

| | キーワード検索 | 位置エミュレーション |
|---|---|---|
| 仕組み | 検索クエリに地名を追加 | X-Geoヘッダーで位置を変更 |
| 例 | `ジム 渋谷駅` | `ジム` (位置: 福岡市) |
| 対応エリア | 路線駅名・区市名 | 47都道府県 + 20政令指定都市 |
| 用途 | 特定の駅・エリアの広告を検索 | 広域の地域ターゲティング広告を取得 |

## 広告検出の仕組み

広告のクリックは一切行わず、検索結果ページのDOMを読み取ってURLを抽出する:

1. `#tads` / `#tadsb` コンテナ内の `[data-text-ad]` 要素から外部リンクを読み取り
2. フォールバック: `aria-label="広告"` 領域から抽出
3. フォールバック: 「スポンサー」ラベルの親要素を辿って抽出

Google広告のクリック計測URL (`/aclk`) は明示的にスキップし、広告主サイトには直接アクセスする。
